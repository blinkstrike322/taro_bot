## 2. О проекте

**Название:** Arcanum.ocv / @amotaro_bot  
**Тип:** Telegram Mini App (WebApp) для гадания на картах Таро  
**Пользователи:** ~200+ активных, RU  
**Хостинг:** Amvera Cloud (контейнер, Python)  
**Бот:** @amotaro_bot  

### Что делает:
1. Пользователь открывает WebApp из Telegram (кнопка в боте)
2. Выбирает тип расклада: 1 карта, 3 карты (Прошлое/Настоящее/Будущее), или Карта Дня
3. Может задать вопрос
4. WebApp показывает карты рубашкой вверх → пользователь тапает → карты переворачиваются с 3D-анимацией
5. LLM (OpenRouter) генерирует интерпретацию в стиле выбранного персонажа-проводника
6. Результат показывается в WebApp
7. История хранится в SQLite, доступен календарь для просмотра прошлых гаданий
8. Бот присылает напоминания раз в 6 часов неактивным пользователям

### 3 персонажа-проводника:
- **Странница Теней** (shadow_walker) — поэтичная, мистическая
- **Хранитель Руин** (ruin_keeper) — прямой, мудрый, без украшений
- **Искра Хаоса** (spark_of_chaos) — дерзкая, с юмором, трюкстер

## 3. Архитектура (два сервера в одном процессе)

```
app.py
  │ asyncio.gather()
  ├── aiohttp на :8080      ← веб-сервер (API + статика)
  ├── aiogram polling       ← Telegram bot (long polling)
  └── reminder_loop         ← напоминалки
```

### Python backend (aiohttp + aiogram):
- **aiohttp** порт 8080: раздаёт статику Next.js + API эндпоинты
- **aiogram**: поллинг Telegram, команды /start, выбор персонажа, WebApp кнопки
- **SQLite** (aiosqlite): таблицы users и readings

### API эндпоинты:
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/spread` | Расклад 1 или 3 карты + интерпретация |
| GET | `/api/readings?init_data=...&year=&month=` | История раскладов за месяц |

### Frontend (Next.js):
- **Next.js 15** + **React 19** + **TypeScript**
- **Tailwind CSS 4**
- **Static export** → `static/webapp/` (aiohttp раздаёт как статику)
- Сборка: `cd web && npm run build`
- Страницы на Pages Router (не App Router)

## 4. Текущий дизайн (что есть сейчас)

### Тема: монохромный CRT-терминал / оккультный панк

**Цвета:** `#000000` (фон), `#FFFFFF` (текст/бордеры), `#666666` (disabled)
**Шрифты:** Pixelify Sans (заголовки), VT323 (CRT-текст), Press Start 2P (бренд), Times New Roman (названия карт — в DESIGN.md, но не используется сейчас)
**Характер:** пиксель-арт, дизеринг, CRT-эффекты (сканлайны, бегущая полоса), глитч-анимации

### Экраны WebApp:
1. **Welcome** — терминал-загрузка (прогресс-бар, `[OCGV_V1.0]`)
2. **Spread input** — поле ввода вопроса + кнопка
3. **Cards** — 1 или 3 карты рубашкой вверх
4. **Result** — интерпретация после переворота всех карт
5. **Модалки:** Каталог раскладов, Настройки (выбор проводника), Календарь (история)

### Что уже есть в UI:
- CRT-оверлей (сканлайны + бегущая полоса через `::before`/`::after`)
- Анимированный шум/статический фон (SVG turbulence)
- Процедурная рубашка карты (CSS-only: звёзды, гексаграммы, розы)
- 3D flip-анимация карт (rotateY, perspective)
- Аура-частицы вокруг карт (unicode символы с комбинирующими диакритиками)
- CursedFooter (Zalgo-текст) — закомментирован
- Плавающие оккультные символы на фоне

## 5. Проблемы текущего дизайна (что улучшать)

1. **Сейчас дизайн НЕ соответствует DESIGN.md** — в доке написано строгий B&W без эффектов, но в коде есть:
   - CRT-эффекты (сканлайны, бегущая полоса, искажение)
   - Цветные glow-эффекты (purple `#6E44FF`, gold `#9E7A35`)
   - Аура-частицы вокруг карт
   - Плавающие символы на фоне
   - Цветные квадратики персонажей (фиолетовый, золотой, красный)
   - Noise-текстура (SVG turbulence)
   - Circuit trace декор (фиолетовые линии)
   - Шрифты: Press Start 2P, VT323 (не указаны в DESIGN.md)

2. **Дизайн feels "busy"** — слишком много визуального шума (сканлайны + глитч + символы + шум + аура + всё сразу)

3. **Нет консистентной типографической системы** — размеры шрифтов разбросаны (6px, 7px, 9px, 10px, 11px, 12px, 13px, 14px, 16px, 18px)

4. **Нет адаптивности для разных устройств**

5. **Модалки не имеют единого стиля** — CatalogModal, SettingsModal, CalendarModal слегка отличаются

6. **Кнопки в Layout имеют разные стили** — primary (bg-white text-black) vs secondary (transparent)

7. **DESIGN.md описывает одно, код реализует другое** — есть расхождение между спецификацией и реализацией

## 6. Технические ограничения (важно!)

### Telegram WebApp:
- Всё внутри `Telegram.WebApp` — инициализация через `window.Telegram.WebApp.initData`
- Telegram требует HTTPS для WebApp (через туннель или Amvera)
- HapticFeedback доступен
- WebApp открывается с параметром `?type=daily|1|3`

### Next.js Static Export:
- `output: 'export'` — **НЕТ серверных функций**, всё клиентское
- `trailingSlash: true`
- `distDir: '../static/webapp'`
- Страницы в `pages/` (Pages Router)
- **Нет API routes** — API на Python/aiohttp, отдельно
- Изображения карт: `/cards/{id}.png` (лежат в `static/webapp/cards/`)

### Сборка:
```bash
cd web
npm install
npm run build   # → static/webapp/
```

### API вызовы:
- Те же origin (относительные URL)
- initData передаётся в теле запроса
- Типы ответов: `SpreadResponse`, `TarotCardData`, `Interpretation`
- Интерпретация приходит JSON-ом: `{ intro, short_answer, card_meaning, advice }`

## 8. Prompt для LLM (что просить)

### Формулировка задачи:

```
Ты — дизайн-революционер. Тебе даётся полный код существующего Telegram WebApp для гадания на Таро.

Задача:
1. Проанализируй весь код, включая DESIGN.md, все компоненты, CSS, архитектуру
2. Предложи РАДИКАЛЬНО улучшенный дизайн в ТОМ ЖЕ стиле (оккультный/мистический/монохромный), но:
   - Избавься от визуального шума (сканлайны, аномалии, лишние эффекты)
   - Приведи к единой типографической системе
   - Сделай более современным и "дорогим"
   - Можешь добавить 1 акцентный цвет (не яркий, а приглушённый)
   - Улучши читаемость и user flow
3. Дай конкретные изменения по каждому компоненту
4. Опиши новую цветовую палитру, типографику, отступы
5. Если нужно — переструктурируй компоненты

ВАЖНО: Все изменения должны:
- Сохранить статический экспорт Next.js (нет server components/server functions)
- Сохранить все существующие API вызовы (api.ts не менять)
- Сохранить все TypeScript интерфейсы (TarotCard, Interpretation, SpreadResponse)
- Сохранить все пропсы существующих компонентов (изменения только внутри)

## 9. Как LLM должна сохранить результат

**КРИТИЧЕСКИ ВАЖНО:** LLM должна сохранить результат так, чтобы его можно было передать Sisyphus для реализации.

### Формат: Git-стиль

LLM должна выдать **для каждого изменяемого файла** одно из:
1. **Полное новое содержимое** (для файлов, которые меняются полностью)
2. **Diff в формате «заменить oldString → newString»** (для частичных изменений)

### Структура ответа:

```
## Файлы для изменения

### [путь к файлу 1] — Полная перезапись

```

```
на:

\`\`\`tsx

## 10. Идеи для нового дизайна (опционально, для размышления)

### Направления, которые можно рассмотреть:

1. **Dark Mystic** — уйти от CRT-пикселей в сторону более чистого мистического минимализма. Тёмный, но не гранж. Как Diablo UI или Dark ACG.

2. **Cyber Occult** — сохранить пиксель-арт и CRT, но сделать более дисциплинированно. Как журнал 2600 или хакерская эстетика.

3. **Grimoire** — дизайн в стиле магического фолианта. Пергаментно-чёрный, обрамления, виньетки, иллюминированные буквицы. Больше типографики, меньше эффектов.

4. **Minimal Dark** — максимально чистый тёмный дизайн. Только типографика, никаких эффектов. Как Arc Browser или Linear в тёмном режиме.

### Что НЕ надо делать:
- Не добавлять яркие неоновые цвета
- Не делать градиенты
- Не уходить в "@media prefers-color-scheme" — только тёмная тема
- Не добавлять анимации ради анимации
- Не создавать новые npm-зависимости

## 11. Процесс: как это будет внедряться

1. **Ты** (человек) копируешь этот документ + содержимое файлов из секции 7
2. Отправляешь LLM (Claude 5.2)
3. LLM анализирует и выдаёт новый дизайн + конкретные изменения
4. Ты возвращаешь результат сюда, в этот чат
5. **Sisyphus** берёт изменения и имплементирует их по файлам
6. Sisyphus запускает `npm run build` чтобы пересобрать WebApp
7. Sisyphus проверяет что всё работает

### Если хочешь быстрее:
Просто скажи: **"Sisyphus, возьми результат из чата и имплементируй все изменения согласно формату из context pack"**
